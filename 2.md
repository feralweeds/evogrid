# 文档二：2D 环境设计文档

## 环境名称

暂定名称：

**EvoGrid-Mine**

或：

**Environment-Shaping Grid World**

## 一、设计目标

本环境用于研究智能体是否能够在 2D 网格世界中通过主动改变环境来提升长期任务表现。

环境设计的核心不是复杂性，而是可控性和可解释性。第一版环境应尽量简单，但必须满足以下条件：

1. 智能体可以改变环境；
2. 改变环境存在短期成本；
3. 改变后的环境会影响未来行动成本；
4. 智能体可以通过训练学会长期策略；
5. 环境变化可以被记录、可视化和消融。

## 二、基本设定

环境是一个二维网格地图，大小可设为：

```text
16 × 16
32 × 32
64 × 64
```

第一版建议使用：

```text
32 × 32
```

地图中包含基地、矿区、障碍物、高成本地形、普通地面和道路。智能体需要在固定时间步内采集矿石并运回基地。

每个 episode 的最大步数可以设置为：

```text
500 steps
```

或：

```text
1000 steps
```

第一版建议从 500 steps 开始。

## 三、地图元素

### 1. Base：基地

基地是智能体出生点，也是资源交付点。

智能体将矿石带回基地后获得奖励。

### 2. Ore：矿物

矿物分布在地图中的若干区域。智能体可以通过 `MINE` 动作采集矿石。

矿物可以设置为有限资源，也可以在第一版中保持固定数量。

### 3. Ground：普通地面

普通地面可以正常通行。

### 4. Obstacle：障碍物

障碍物默认不可通行。智能体可以使用 `DIG` 动作将障碍物变为可通行地面。

### 5. Rough Terrain：高成本地形

高成本地形可以通行，但移动成本更高，或者需要消耗更多时间步。

### 6. Road：道路

道路由智能体通过 `BUILD_ROAD` 动作建造。道路可以降低移动成本，或者让智能体在上面移动时减少能量消耗。

### 7. Optional Depot：中转仓库

第一版可以暂不实现。第二版可以加入仓库，使智能体能够将矿石暂存在中间地点，从而改变资源运输结构。

## 四、状态表示

环境状态包括：

```text
map: H × W 的地图矩阵
agent_position: 智能体当前位置
agent_inventory: 智能体是否携带矿石
time_step: 当前步数
changed_map_mask: 哪些格子被智能体改造过
visited_mask: 哪些格子被智能体访问过
```

如果使用深度强化学习，可以将状态编码为多通道图像：

```text
Channel 1: base
Channel 2: ore
Channel 3: obstacle
Channel 4: rough terrain
Channel 5: road
Channel 6: agent position
Channel 7: changed map mask
```

如果使用 tabular 或简单策略，可以先使用低维特征表示。

## 五、动作空间

第一版建议使用离散动作空间。

```text
0 = MOVE_UP
1 = MOVE_DOWN
2 = MOVE_LEFT
3 = MOVE_RIGHT
4 = MINE
5 = DIG
6 = BUILD_ROAD
7 = DROPOFF
8 = NOOP
```

### MOVE

智能体向指定方向移动一格。
如果目标格是障碍物，则移动失败。
如果目标格是高成本地形，可以消耗额外时间或给予额外惩罚。
如果目标格是道路，则移动成本降低。

### MINE

如果智能体当前位置或相邻位置存在矿物，则采集矿物。
第一版建议规定智能体只能携带一单位矿石。

### DIG

如果智能体相邻位置是障碍物，则可以将其挖开，变为普通地面。
`DIG` 应有一定成本，例如消耗多个时间步或给予负奖励。

### BUILD_ROAD

智能体可以在当前位置建造道路。
道路会改变地图，使后续经过该格子的移动成本降低。

### DROPOFF

如果智能体位于基地且携带矿石，则交付矿石并获得奖励。

### NOOP

智能体不执行任何动作。
该动作用于处理等待、无效状态或策略稳定性。

## 六、奖励函数

奖励设计应避免直接奖励环境改造行为。
否则智能体可能只是为了奖励而修路，而不是为了长期收益。

建议第一版奖励函数如下：

```text
成功交付一单位矿石：+10
每一步行动成本：-0.01
DIG 成本：-0.2
BUILD_ROAD 成本：-0.1
无效动作：-0.05
episode 结束未交付资源：无额外奖励
```

核心原则：

> 修路和挖掘本身应该有成本，但它们可以通过降低未来运输成本间接提高长期收益。

## 七、环境动态

### 1. 道路降低移动成本

最简单的实现方式：

普通地面移动成本：

```text
-0.01
```

道路移动成本：

```text
0
```

或：

普通地面移动需要 1 step。
高成本地形移动需要 2 steps。
道路移动只需要 1 step。

如果环境实现不方便支持变长时间步，可以用奖励惩罚表示移动成本差异。

### 2. 挖掘改变可达性

障碍物默认不可通行。
智能体使用 `DIG` 后，障碍物变成普通地面。
这样智能体可以通过挖通道缩短基地和矿区之间的路径。

### 3. 修路改变未来策略

道路一旦建成，在当前 episode 内持续存在。
第二阶段可以考虑跨 episode 保留道路，但第一版建议先只在 episode 内保留，方便证明学习发生在策略中，而不是仅仅发生在地图积累中。

### 4. 地图随机生成

第一版可以使用固定地图，方便调试。
第二版需要随机生成地图，用于验证泛化能力。

地图生成可以包括：

```text
固定基地位置
随机矿区位置
随机障碍物分布
随机高成本区域
保证基地到矿区至少存在一条可达路径
```

## 八、训练目标

智能体目标是在有限时间内最大化累计奖励。

理想学习过程如下：

### 训练早期

智能体随机探索，可能偶尔采矿，但路径混乱，修路行为随机，资源交付效率低。

### 训练中期

智能体开始发现矿区和基地之间的反复运输需求，偶尔在路径上修路或挖障碍。

### 训练后期

智能体学会在任务早期投入部分时间修路或挖通道，然后在后期高效运输矿石。

## 九、可视化设计

为了展示“学习”和“环境塑造”，需要至少实现以下可视化：

### 1. 原始地图渲染

显示基地、矿区、障碍物、道路和智能体位置。

### 2. Before / After 地图对比

展示 episode 开始时的地图和结束时的地图。

重点观察：

```text
哪些地方被修路
哪些障碍被挖开
道路是否连接基地和矿区
```

### 3. 轨迹热力图

记录智能体经过每个格子的次数。
训练后期应该出现更集中的主干路径。

### 4. 改造热力图

记录被 `DIG` 或 `BUILD_ROAD` 改造过的格子。
有效的环境改造应该集中在关键路径，而不是随机分布。

### 5. 训练曲线

至少包括：

```text
累计奖励
交付矿石数量
道路使用率
平均运输路径长度
改造格子数量
```

## 十、消融设置

为了证明智能体不是单纯依靠环境机制，而是真的学会了环境塑造，需要设计以下对照：

### Full Shaping

允许 `DIG` 和 `BUILD_ROAD`。

### No Shaping

移除或禁用 `DIG` 和 `BUILD_ROAD`。

### Reset Shaping

允许智能体改造环境，但每次交付矿石后清除道路和挖掘痕迹。

### High Cost Shaping

提高修路和挖掘成本，观察智能体是否减少建设行为。

### Random Agent

使用随机策略，验证随机改造无法带来同样收益。

## 十一、第一版最小实现范围

第一版只需要实现：

```text
32 × 32 grid map
1 个智能体
1 个基地
若干矿物
障碍物
道路
MOVE / MINE / DIG / BUILD_ROAD / DROPOFF
累计奖励
地图渲染
训练曲线
Full vs No Shaping 消融
```

暂不实现：

```text
多智能体
竞价机制
仓库
桥梁
塌方
跨 episode 持久化
复杂 crafting
```

## 十二、后续扩展方向

后续可以加入：

1. 多智能体共享环境；
2. 固定角色：builder、miner、carrier；
3. 自然分工；
4. 仓库和运输链；
5. 道路维护成本；
6. 资源枯竭和新矿区发现；
7. 跨 episode 持久化世界；
8. 基于 LLM 的高层规划；
9. 与 Craftax-Classic 对接；
10. 多智能体任务分配或竞价机制。

## 十三、总结

该环境的核心价值在于提供一个简洁、可控、可消融的平台，用于验证：

> 智能体是否能够学会通过改变环境来改善未来任务条件。

环境不需要复杂，但必须清楚表达“短期建设成本”和“长期任务收益”之间的关系。
