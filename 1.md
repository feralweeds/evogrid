# 文档一：项目研究方案

## 项目名称

**环境塑造型智能体：在可改造 2D 世界中通过改变环境进行学习**

英文暂定名：

**Environment-Shaping Agents: Learning by Modifying a Persistent 2D World**

## 一、研究背景

当前主流智能体研究通常将环境视为一个相对固定的外部系统。智能体通过与环境交互，获得观测、执行动作、接收奖励，并在此过程中优化自身策略。这类方法在强化学习、多智能体协作、工具使用和开放世界探索中已经取得了大量进展。

然而，在许多真实场景中，智能体并不是单纯适应环境，而是会主动改变环境。例如，机器人在矿洞中挖掘通道、建造支撑结构；人类在城市中修路、建桥、建立仓储系统；多智能体系统中，不同智能体可能通过协作建设基础设施，从而改变未来任务的难度和收益结构。

因此，本项目关注一种更进一步的智能体学习范式：

> 智能体是否能够通过主动改造环境，创造更有利于自身未来行动和学习的状态分布？

本项目将该方向称为**环境塑造型智能体**，也可以理解为“在改变环境中学习”的智能体系统。

## 二、核心研究问题

本项目的核心问题是：

> 智能体能否学会为了长期收益而主动改变环境，并在被自己改变过的环境中获得更高的任务表现？

围绕这一问题，进一步拆分为以下几个子问题：

1. 智能体是否会在没有直接奖励的情况下执行环境改造行为？
2. 智能体是否会从短视任务策略转向长期基础设施建设策略？
3. 被智能体改造后的环境是否能够降低后续任务成本？
4. 如果禁止环境改造，智能体表现是否显著下降？
5. 如果清除环境改造痕迹，智能体是否失去部分性能优势？
6. 在多智能体版本中，是否会出现建设、采集、运输等角色分化？

## 三、项目目标

本项目的第一阶段目标是构建一个轻量级 2D 可改造环境，并训练智能体在其中完成资源采集任务。

环境中包含基地、矿区、障碍、高成本地形、道路等元素。智能体需要在有限时间内采集资源并将其运回基地。与普通采集任务不同的是，智能体可以通过修路、挖通道等方式改变环境结构，从而降低未来采集和运输成本。

第一阶段暂不将竞价机制作为核心内容。竞价机制可以作为后续多智能体任务分配的可选扩展。

## 四、研究假设

本项目提出以下假设：

### 假设 1：环境改造可以提升长期收益

允许智能体修路或挖通道后，智能体在长期任务中的累计收益应高于禁止环境改造的智能体。

### 假设 2：智能体能够学习到非短视策略

如果修路和挖通道在短期内存在成本，但长期能够提升运输效率，那么训练后的智能体应学会在任务早期投入一定成本进行建设，并在后期获得更高收益。

### 假设 3：环境改造痕迹具有功能性

智能体改造过的道路、通道或基础设施应该被后续高频使用，而不是随机分布在地图上。

### 假设 4：环境塑造策略具有一定泛化能力

训练后的智能体在未见过的新地图中，仍应能够主动识别关键路径并进行有效环境改造。

## 五、拟采用环境

第一版建议采用两种实现路线之一：

### 路线 A：自研极简 Gym 环境

优点是实现简单、可控性高、便于快速验证想法。
缺点是与已有 benchmark 的连接较弱。

### 路线 B：基于 Craftax-Classic 进行轻量扩展

优点是已有 2D open-world、生存、采集、制作和建造机制，训练速度快，后续更容易向论文 benchmark 靠拢。
缺点是 Craftax 基于 JAX，二次开发难度高于普通 Python Gym 环境。

项目第一版建议优先做一个极简环境进行机制验证；如果时间允许，再迁移到 Craftax-Classic。

## 六、环境核心机制

环境采用 2D 网格世界。地图中包含：

* 基地；
* 矿区；
* 普通地面；
* 高移动成本地形；
* 障碍物；
* 道路；
* 被挖开的通道；
* 可选的仓库或中转点。

智能体动作包括：

* 上下左右移动；
* 采矿；
* 挖掘障碍；
* 修建道路；
* 交付资源；
* 空操作。

其中，修路和挖掘属于**环境塑造动作**。这些动作本身不提供直接奖励，甚至会产生时间或资源成本，但会改变地图结构并影响后续任务效率。

## 七、实验设计概述

主要实验组包括：

1. **Full Shaping**：允许智能体修路、挖通道等环境改造行为。
2. **No Shaping**：禁止所有环境改造动作。
3. **Reset World**：允许环境改造，但每轮清除改造痕迹。
4. **High Cost Shaping**：提高修路或挖掘成本。
5. **Random Agent**：随机策略作为下界对照。
6. **Trained Agent**：训练后策略作为主要比较对象。

通过这些实验，可以验证智能体是否真正学会了利用环境改造，而不是仅仅依赖环境本身的设定。

## 八、评价指标

项目主要评价指标包括：

1. 累计奖励；
2. 资源采集总量；
3. 单位资源运输成本；
4. 基地到矿区的平均路径长度；
5. 道路使用率；
6. 改造格子数量；
7. 有效改造比例；
8. 前期建设投入与后期收益提升；
9. 新地图泛化表现。

其中，最关键的不是单纯奖励上升，而是要证明：

> 智能体学会了通过短期环境改造换取长期任务收益。

## 九、预期成果

第一阶段预期完成：

1. 一个可运行的 2D 可改造环境；
2. 一个支持训练的 RL 接口；
3. 至少一个 baseline 智能体；
4. 训练曲线；
5. 环境改造前后对比图；
6. 智能体轨迹热力图；
7. Full vs No Shaping 的消融实验；
8. 一个可展示的 demo 视频或交互界面。

第二阶段可扩展：

1. 多智能体共享环境；
2. 固定角色或自然分工；
3. 仓库、桥梁、支撑柱等基础设施；
4. 跨 episode 的持久化世界；
5. 多智能体任务分配机制；
6. 与 Craftax / Minecraft-like 环境连接。

## 十、项目定位

本项目不以“构建一个复杂 2D 游戏”为核心目标，而是以提出和验证一个清晰的研究问题为主：

> 环境是否可以成为智能体学习和进化过程的一部分？

如果实验能够证明智能体会主动改造环境，并利用这些改造提升未来表现，则可以为“自进化智能体”提供一个可控、可复现、可消融的实验平台。
