# 文档三：实验评估方案

## 一、实验目标

本实验的目标是验证智能体是否能够在可改造 2D 环境中学会环境塑造策略。

具体而言，我们希望证明：

1. 智能体能够通过训练提升任务表现；
2. 智能体会主动执行修路、挖掘等环境改造行为；
3. 这些环境改造行为不是随机的，而是具有功能性；
4. 环境改造能够降低未来任务成本；
5. 禁止或清除环境改造会导致性能下降；
6. 训练后的策略在新地图上仍具有一定泛化能力。

## 二、主要实验假设

### 假设 1：允许环境改造可以提高长期收益

与禁止环境改造的智能体相比，允许修路和挖掘的智能体应获得更高累计奖励和更多资源交付量。

### 假设 2：智能体会学习长期建设策略

如果修路和挖掘具有短期成本，训练后的智能体仍然主动执行这些行为，则说明其策略不只是短视采矿，而是学会了以短期成本换取长期收益。

### 假设 3：有效环境改造会集中在关键路径

训练后的智能体修建的道路或挖出的通道应该主要位于基地和矿区之间的高频路径上，而不是随机分布。

### 假设 4：清除改造痕迹会损害表现

如果清除道路和通道后，智能体表现明显下降，则说明环境改造本身对任务完成有实质贡献。

## 三、实验环境

实验环境为 2D 网格世界。

默认配置：

```text
地图大小：32 × 32
智能体数量：1
基地数量：1
矿区数量：1 至 3
最大步数：500
动作空间：MOVE、MINE、DIG、BUILD_ROAD、DROPOFF、NOOP
```

环境中包含：

```text
基地
矿物
普通地面
障碍物
高成本地形
道路
智能体
```

## 四、实验组设置

### 1. Full Shaping

允许智能体使用完整动作空间，包括：

```text
DIG
BUILD_ROAD
```

这是主要实验组。

### 2. No Shaping

禁止环境改造动作。

智能体只能：

```text
MOVE
MINE
DROPOFF
NOOP
```

该组用于验证环境改造是否带来额外收益。

### 3. Road Only

只允许 `BUILD_ROAD`，不允许 `DIG`。

用于分析修路机制的独立作用。

### 4. Dig Only

只允许 `DIG`，不允许 `BUILD_ROAD`。

用于分析挖通道机制的独立作用。

### 5. Reset World

允许智能体修路和挖掘，但在每次交付矿石后，清除本轮环境改造痕迹。

用于验证智能体是否依赖持续存在的环境结构。

### 6. High Cost Shaping

提高修路和挖掘成本。

例如：

```text
BUILD_ROAD: -0.5
DIG: -1.0
```

用于观察环境改造成本对策略的影响。

### 7. Random Agent

随机选择动作。

用于证明随机环境改造不能带来同等收益。

### 8. Greedy Agent

手写贪心策略，只追求最近矿石，不主动修路或挖通道。

用于比较训练智能体是否超越短视策略。

## 五、训练算法

第一版建议使用 PPO 或 DQN。

如果环境是图像状态：

```text
PPO + CNN policy
```

如果环境状态较简单：

```text
PPO + MLP policy
```

第一版目标不是追求最强算法，而是验证机制。因此算法选择应优先考虑稳定、易实现、易复现。

建议 baseline：

```text
PPO
Random
Greedy
```

如果时间允许，可以加入：

```text
DQN
A2C
Rule-based planner
```

## 六、奖励函数

建议奖励设置：

```text
交付矿石：+10
普通移动：-0.01
高成本地形移动：-0.05
DIG：-0.2
BUILD_ROAD：-0.1
无效动作：-0.05
```

注意：

不直接奖励修路。
不直接奖励挖通道。
修路和挖通道只通过改善未来行动效率间接带来收益。

这一点非常重要，因为项目要证明的是：

> 智能体为了长期收益主动改造环境，而不是为了即时奖励改造环境。

## 七、核心评价指标

### 1. 累计奖励

每个 episode 的总 reward。

用于判断智能体整体表现是否提升。

### 2. 资源交付数量

智能体成功带回基地的矿石数量。

这是最直观的任务表现指标。

### 3. 平均运输成本

定义为：

```text
从基地到矿区再返回基地的平均步数或平均负奖励成本
```

如果环境改造有效，该指标应随训练下降。

### 4. 道路使用率

定义为：

```text
被智能体走过的道路格子数 / 智能体建造的道路格子数
```

道路使用率越高，说明修路越有目的性。

### 5. 改造有效率

定义为：

```text
位于基地-矿区最短路径附近的改造格子数 / 总改造格子数
```

用于衡量环境改造是否集中在关键区域。

### 6. 前期建设比例

定义为：

```text
前 K 步中环境改造动作数量 / 前 K 步总动作数量
```

用于观察智能体是否学会在早期投资建设。

### 7. 后期收益提升

将一个 episode 分为前半段和后半段，比较后半段单位时间资源收益。

如果智能体前期修路、后期高效采矿，则后期收益应明显提高。

### 8. 新地图泛化表现

在训练未见过的地图上测试智能体表现。

如果训练后的智能体仍能主动修路和挖通道，说明它学到的是策略，而不是记住某张地图。

### 4. 道路使用率曲线

横轴：训练步数

展示训练前和训练后的环境改造情况。

### 6. 轨迹热力图

### 7. 改造热力图

## 九、关键消融实验

### 消融 1：禁止环境改造

比较：

```text
Full Shaping vs No Shaping
```

若 Full Shaping 表现更好，说明环境改造机制有效。

### 消融 2：清除环境改造痕迹

比较：

```text
Full Shaping vs Reset World
```

若 Reset World 表现下降，说明持续存在的环境结构对任务有帮助。

### 消融 3：提高环境改造成本

比较：

```text
Low Cost Shaping vs High Cost Shaping
```

若成本提高后，智能体减少修路或只在关键路径修路，说明其行为对成本敏感，具有策略性。

### 消融 4：新地图测试

比较训练地图和测试地图表现。

若新地图表现仍优于 No Shaping，说明智能体学到了可泛化的环境塑造策略。

## 十、预期现象

理想情况下，训练过程应出现以下阶段：

### 训练早期

```text
智能体随机探索
经常无效移动
修路和挖掘位置随机
资源交付数量低
```

### 训练中期

```text
智能体开始找到矿区
偶尔修路或挖通道
运输路径开始集中
奖励逐渐上升
```

### 训练后期

```text
智能体前期主动建设
基地和矿区之间出现稳定道路或通道
后期资源运输效率明显提高
累计奖励稳定上升
```

## 十一、失败情况与解释

如果智能体没有学会修路，可能原因包括：

1. 修路成本过高；
2. episode 太短，长期收益无法体现；
3. 道路收益太弱；
4. 地图太简单，不需要修路；
5. 奖励过于稀疏；
6. 算法探索能力不足；
7. 动作空间过大，学习难度过高。

对应调整方式：

```text
降低 BUILD_ROAD 成本
增加 episode 长度
提高道路收益
增加障碍和高成本地形
加入中间奖励但避免直接奖励修路
使用 curriculum learning
简化动作空间
```

## 十二、Demo 展示方案

最终 demo 建议展示四个部分：

### 1. 训练前后对比视频

展示训练早期和训练后期智能体行为差异。

### 2. 地图变化对比

展示 episode 开始和结束时地图变化。

### 3. 轨迹热力图

展示智能体路径从随机游走变成稳定主干路径。

### 4. 消融实验图表

展示 Full Shaping 明显优于 No Shaping 和 Random Agent。

## 十三、最终判断标准

如果实验结果满足以下条件，则可以认为第一阶段成功：

1. Full Shaping 的累计奖励显著高于 No Shaping；
2. 训练后的智能体会主动执行修路或挖掘；
3. 修路和挖掘集中在基地与矿区之间；
4. 道路使用率随训练提高；
5. 清除环境改造痕迹后性能下降；
6. 在新地图上仍能部分复现环境塑造行为。

如果以上条件成立，就可以说明：

> 智能体不仅学会了在环境中行动，而且学会了通过改变环境来改善未来行动条件。
