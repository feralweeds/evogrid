## 2. 总体判断

当前版本已经完成较完整的工程原型和 smoke 流程，但尚未完成“智能体自进化”的正式实验闭环。

| 维度 | 当前判断 | 说明 |
| --- | --- | --- |
| 架构与技术文档 | 基本完成 | 已形成架构决策、技术规格、任务分解和验收协议 |
| 自动化测试 | 通过 | `235` 项测试通过，未发现单元测试回归 |
| 工程原型 | 大部分可运行 | M0-M2 较完整，M3-M7 有不同程度的接口或 smoke 实现 |

## 4. 里程碑完成情况

状态定义：

- **通过**：已满足该里程碑退出条件；
- **条件通过**：核心实现存在，但还需补充明确的发布证据；
- **部分完成**：只有部分契约或接口可用；
- **E0 smoke**：只验证流程，不支持该里程碑的科研结论；
- **未通过**：不得作为后续正式实验依赖。

| 里程碑 | 工程完成度估计 | 验收状态 | 主要依据 |
| --- | ---: | --- | --- |
| M0 契约与兼容层 | 90% | 条件通过 | 稳定 Seed、配置 schema、兼容地图入口和测试已存在 |
| M1 有限参数化地图 | 80% | 未通过正式 Gate | 分形场、相关渗流、放置和诊断已实现；A0-A5 尚未正式验收 |
| M2 连续地形与因果成本 | 80% | 条件通过 | roughness、移动成本、道路收益和观测隔离已接入；正式 R2 证据不足 |

| M4 自主提出与独立验证 | 40% | 未通过 | 具备配对验证骨架，但统计 Gate 和晋升边界不完整 |

## 5. 阻塞正式验收的问题

### 5.1 P0：M5/M6 使用合成指标，不是真实实验

`scripts/run_skill_evolution_experiment.py` 中的 capability group score 是预设常量，地图诊断和部分效果量也是公式化 fixture。`scripts/run_curriculum_ablation.py` 使用预设 base score 和确定性 jitter 生成消融结果。

影响：

- 不能证明 Prompt 与 Skill 存在真实效果差异；
- 不能证明 Verified Skill Set 随环境阶段增长；
- 不能证明自适应课程促进技能获得或保持；
- 当前图只能作为绘图和产物格式演示。

完成条件：M5/M6 的每一行 episode metric 必须来自真实 `env.reset -> agent.act -> env.step` rollout，并能由 seed、配置和代码版本重放。

### 5.2 P0：Skill Runtime 未完整实现 Procedure DSL

`evogrid/skills/runtime.py` 中 `SELECT_TARGET`、`PLAN_ROUTE`、`FOLLOW_ROUTE` 当前返回 `not_implemented_noop`，`CALL_SKILL` 因无嵌套 Registry 而终止。

影响：

- Skill 不能稳定表达多步路线和复合行为；
- M3 的“执行、重放、组合和消融”退出条件没有满足；
- Candidate 即使通过 schema，也可能没有真实可执行语义。

完成条件：所有白名单操作具有确定语义、预算扣减、非法动作回退、执行 trace 和定向测试；未实现操作不得出现在可验证 Skill schema 中。

### 5.3 P0：Skill Verifier 和 Registry 晋升边界不完整

当前 verifier 主要根据均值、正向比例、运行失败率和误触发率生成 G0-G3；尚未完整实现 bootstrap CI、最小样本、G4、G5 和多候选 FDR。Registry 会依据报告中的 `decision` 改变状态，但没有重新验证所有适用 Gate 的一致性。

影响：伪造、过小样本或内部不一致的报告可能把 Candidate 晋升为 Verified Skill。

完成条件：

1. 正式 stratum 至少 30 个配对 seed；
2. G1 使用预注册 bootstrap CI 和 `min_effect`；
3. G2-G5 按 `04_实验与验收协议.md` 实现；
4. proposal batch 执行 Benjamini-Hochberg FDR；
5. Registry 根据原始指标和版本化协议重新计算 decision；
6. `lease_id`、`spec_hash`、partition 和 protocol hash 全部一致才允许晋升。

### 5.4 P0：R1 只有 smoke，没有 A0-A5 正式判定

现有地图校准运行使用 smoke 参数和少量 seed，没有产出 A0-A5 逐项判定、二项置信区间、有限尺寸 `p50`、轴向伪影比较和正式 H 可分性结果。

影响：不能声称 H、`p_open` 和 size 已经形成可控环境分布，也不能将其作为环境课程的可靠自变量。

完成条件：执行第 7.2 节的 R1 恢复任务，并由机器可读验收报告证明 A0-A5 全部通过。

### 5.5 P1：连续地形不是默认实验路径

`configs/env_fractal_percolation.yaml` 当前使用 `reward_mode: legacy_discrete`。连续 roughness 到移动成本、道路收益和策略选择的因果链主要由单元测试覆盖，尚未成为正式实验配置。

影响：默认实验不会检验导师提出的多尺度地形结构是否改变 Agent 学习压力。

完成条件：保留 legacy 配置用于兼容，新增独立的连续地形正式配置，并完成 R2 数值、因果和性能验收。

### 5.6 P1：Chunk 道路语义与有限世界不一致

`evogrid/envs/chunk_world/world.py` 在应用 `build_road` 事件时会把 `walkable` 同时设为真。规范要求道路只改变移动成本，只有 DIG 才改变通行性。

影响：同一 Skill 在有限地图和 chunk 世界中的动作含义不同，迁移实验无效。

完成条件：统一有限世界和 chunk 世界的事件语义，并增加“道路不打开障碍格”的回归测试。

### 5.7 P1：Acceptance Auditor 不能独立证明结论等级

当前 auditor 主要根据 `experiment_type` 和 smoke 标记推断 E1-E5，没有重新核验 A0-A5、G0-G5 和真实 rollout 来源。

影响：只要 manifest 类型填写错误，就可能高估结论强度。

完成条件：结论等级必须由机器可验证的 Gate 产物推导；缺少原始记录、协议 hash 或 Gate 报告时一律降为 E0 或失败。

## 6. 后续执行原则

1. 先补证据链，再扩展功能范围。
2. smoke、mock、pilot、formal 必须使用不同 run ID 和 manifest 标记。
3. mock 数据不得进入正式汇总图，不得和真实 rollout 合并计算。
4. 每个正式结果必须能追溯到代码 commit、resolved config、seed partition、prompt hash 和 Skill Set hash。
5. Candidate 生成数据、验证数据和最终测试数据必须物理或逻辑隔离。
6. Agent 可见信息、LLM prompt、Skill runtime context 不得包含 audit 真值。
7. reward 只作为行为表现指标；Agent 能力主纵轴必须来自 Verified Skill Set 和固定 capability benchmark。
8. 后续模型不得因为测试通过而自动宣布里程碑通过。

## 7. 推荐恢复顺序

### 7.1 阶段 0：冻结可复现开发基线

目标：把当前可运行原型转化为可追踪基线。

必须完成：

4. 运行 235 项完整测试；
5. 生成 `baseline_audit.json`，记录命令、耗时、通过数和失败数。

退出条件：工作区状态、依赖和测试结果可被另一环境复现。

### 7.2 阶段 1：完成 R1 正式地图校准

目标：证明 `p_open`、H 和 size 是可控环境变量。

实现任务：

1. 新增 `configs/map_calibration_formal.yaml`；
2. 将 topology、terrain 和 resource calibration 分为独立扫描；
3. 扩展校准脚本，输出 A0-A5 的机器可读判定；
4. 实现二项置信区间、经验 H 分组统计、轴向相关差异和 `p50` 估计；
5. 扩展 acceptance auditor，使 E1 由 A0-A5 报告推导。

首个正式矩阵：

```text
size: [64, 128, 256]
topology_hurst: [0.2, 0.5, 0.8]
p_open: [0.40, 0.45, 0.50, 0.55, 0.60, 0.65, 0.70, 0.75, 0.80, 0.85]
```

建议命令形式：

```powershell
python scripts/calibrate_fractal_maps.py `
  --config configs/map_calibration_formal.yaml `
  --out outputs/runs/r1_map_calibration_formal_v1 `
  --seeds 0:100
```

退出条件：A0-A5 全部通过，产物审计给出 E1；任一 Gate 失败时保留原始结果并修复生成器或协议，不得人工删除失败 seed。

### 7.3 阶段 2：完成 R2 连续地形因果链

目标：验证 roughness 通过移动成本和道路收益影响策略，而不是直接改变 reward 标签。

实现任务：

1. 新增连续地形正式配置，保留 legacy 配置；
2. 执行 `04_实验与验收协议.md` 的数值单元测试；
3. 对同图、同 seed 比较道路启用和禁用；
4. 报告移动成本、净道路收益、任务成功率和运行性能；
5. 确认 partial observation 和 prompt 无隐藏 roughness 真值泄漏。

退出条件：R2 全部数值、因果和性能 Gate 通过。

### 7.4 阶段 3：完成 R3 Skill 基础设施

目标：建立 Prompt、Candidate Skill 和 Verified Skill 的不可混淆边界。

执行顺序：

1. 完成或收缩 Procedure DSL；
2. 实现 Skill 执行、重放、禁用和消融；
3. 补齐 verifier 的 G0-G5、bootstrap 和 FDR；
4. 加固 Registry 的报告一致性和租约校验；
5. 使用人工正例 fixture 验证可晋升路径；
6. 使用负例、伪造报告和泄漏上下文验证拒绝路径。

退出条件：只有独立 verifier 能晋升 Candidate，LLM 输出不能直接或间接越权。

### 7.5 阶段 4：用真实 rollout 替换 M5 mock

目标：首次获得环境阶段与 Agent Skill Set 的真实耦合曲线。

每个对照组必须实际运行：

- `no_skill`；
- `prompt_only`；
- `handcrafted_skill`；
- `self_proposed_candidate`；
- `self_proposed_verified`；
- `prompt_skill`。

所有组必须固定环境预算、primitive action budget、LLM action-call budget、base prompt hash 和 seed partition。不得读取当前脚本中的预设 group score。

退出条件：episode metrics 全部来自真实 rollout；能力曲线由固定未见 benchmark 计算；结论严格限制在实际达到的 E0-E4。

### 7.6 阶段 5：恢复 M6 自适应课程

目标：判断环境调整是否促进技能获得、保持和迁移。

进入条件：M5 已得到可复现的正结果或明确负结果。

必须比较固定阶段、随机参数、自适应课程、快速晋升和静态单环境。Controller 只能读取 train/gate，不得读取 verify/test。

退出条件：课程消融使用真实 rollout，环境调整事件可重放，结论满足 E5 所需证据。

### 7.7 阶段 6：恢复 M7 Chunk 开放世界

目标：把已经在有限世界验证的环境统计和 Skill 语义扩展到可持久化大地图。

必须先修复：

1. 道路和 DIG 事件语义；
2. chunk 与有限地图的 terrain band 命名；
3. 全局场的经验 H 和实际开放比例校准；
4. Agent、Memory、Skill runtime 与 chunk observation 的正式集成。

退出条件：跨 chunk 连续、加载顺序无关、事件持久化、缓存有界，以及有限地图 Verified Skill 的未见区域迁移全部通过。

## 10. 最终验收边界

当前可以对外表述：

> 已完成参数化环境、连续地形、Skill 生命周期和 chunk 世界的工程原型，自动化测试与 smoke 流程可运行；正在进行正式地图统计校准和 Skill 独立验证机制建设。

当前不得对外表述：

- Agent 已经自主形成有效 Skill；
- Skill Set 已经发生持续进化；
- Hurst 或渗流参数已经被正式证明可控；
- 自适应环境促进了智能体自进化；
- chunk 开放世界中的技能迁移已经成立。

只有当对应的 A/G Gate、真实 rollout、未见 seed 和审计产物全部通过后，才能提升结论等级。
