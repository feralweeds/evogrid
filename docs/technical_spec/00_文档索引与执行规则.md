# EvoGrid 智能体自进化平台：文档索引与执行规则

版本：1.2
状态：生产指导基线  
上游需求：`docs/7.14计划.md`、`docs/EvoGrid_今日讨论总结_2026-07-22.md`
适用对象：负责实现、测试、实验运行和结果审计的底层模型或工程人员

## 1. 结论

`docs/7.14计划.md` 与导师建议总体一致，可以作为研究方向文档。它准确覆盖了以下主线：

1. 从小型受控地图扩展到局部观测、持久改造的大型环境；
2. 用 Hurst 指数控制空间相关结构；
3. 用渗流参数控制地图连通性；
4. 研究环境变化与 Agent 能力增长的耦合；
5. 用经过验证的 Skill Set 而不是单次 reward 表示 Agent 能力；
6. 区分 Prompt、经验记录、Candidate Skill 和 Verified Skill；
7. 让 Agent 经历 `Propose -> Solve -> Verify -> Store/Revise/Delete` 的技能生命周期。

计划仍属于研究设计，不足以直接指导生产实现。本目录补齐参数语义、模块边界、数据契约、统计门槛、实施顺序和验收条件。

文档冲突时，优先级从高到低为：

1. 本目录中编号更小的规范性约束；
2. `02_可执行技术规格书.md` 中的数据契约；
3. `04_实验与验收协议.md` 中的科学有效性约束；

6. `06_R4_Postmortem_and_Positive_Fixture.md` 和 `05_任务完成情况评估与后续实施指南.md` 中的动态审计说明；

发现无法消解的冲突时，停止相关模块的实现，记录冲突位置，不得自行扩大研究结论。

## 3. 规范性用词

- **必须**：验收不可缺少；不满足即任务失败。
- **不得**：明确禁止；实现中不得以便利为由绕过。
- **应该**：默认要求；只有在记录理由和替代验证后才能偏离。
- **可以**：可选实现，不影响当前里程碑验收。

## 4. 生产执行规则

### 4.1 兼容性

1. 现有固定地图、随机课程和受控走廊实验必须继续运行。
2. 新地图生成器通过新的 `map_mode` 接入，不得改写旧生成器的统计分布。
3. 旧配置缺少新字段时必须采用旧行为，不得静默切换到连续地形奖励。
4. 输出 JSON/CSV 必须带 `schema_version`；新增字段优先保持向后兼容。

### 4.2 科学有效性

1. partial observation 下不得向 Agent、LLM prompt、Skill runtime 暴露隐藏地图真值。
2. 渗流校准地图不得通过人工开路或反复重采样来强制可达。
3. 相关渗流的临界点必须由当前生成器和有限尺寸实验估计，不得直接套用独立方格渗流常数。
4. Candidate Skill 不得因为 LLM 声称“有效”而晋升；只有独立验证器可以改变其验证状态。
5. Prompt 内容变化和 Skill Set 变化必须分别记录并分别做消融。
6. reward 是验证信号之一，不是 Agent 能力的唯一纵轴。

### 4.3 安全执行

1. LLM 不得生成并执行任意 Python、shell、SQL 或动态导入代码。
2. Skill 只能由版本化、可校验的声明式 DSL 表示，并由受限 runtime 执行。
3. 所有 Skill 执行必须有步数预算、终止条件和非法动作回退。
4. API key、完整 prompt 中的密钥和个人信息不得写入输出。

### 4.4 工程范围

1. 每个任务包只修改其声明的模块；不得顺便重构无关代码。
2. 每完成一个任务包，先运行其定向测试，再运行完整测试。
3. 不得删除或覆盖用户已有输出、文档和未提交改动。
4. 不得先做 chunk 无限世界，再补有限地图统计校准。
5. 不得先做自适应课程，再证明固定参数环境族可控。

## 5. 里程碑顺序

| 里程碑 | 目标 | 进入条件 | 退出条件 |
| --- | --- | --- | --- |
| M0 | 契约与兼容层 | 当前测试可运行 | 新配置、数据类和 schema 测试通过 |
| M1 | 有限参数化地图 | M0 完成 | `p_open`、H、size 的统计校准通过 |
| M2 | 连续地形与因果成本 | M1 完成 | 移动成本、修路归因、局部观测通过 |
| M3 | Skill 数据层与受限 runtime | M2 完成 | Skill 可存储、调用、消融、重放 |
| M4 | Propose/Verify 生命周期 | M3 完成 | Candidate 只能经独立验证晋升 |
| M5 | 固定环境族和课程实验 | M4 完成 | Prompt/Skill 对照和能力曲线可复现 |
| M6 | 自适应环境课程 | M5 有正结果或明确负结果 | 难度调整不泄露测试集 |
| M7 | Chunk 开放世界 | M1-M6 的有限世界证据稳定 | 跨 chunk 连续、顺序无关、持久化通过 |

底层模型默认只实施当前被明确授权的里程碑。完成文档不代表已经授权一次性实现 M0-M7。

2026-07-22 后的当前授权执行入口是 `WP-R5-00`。R5 的目标是补齐最小通用路线目标选择能力、真实环境人工 road positive skill、有限 LLM 回归和 road 主线停止审计；不得继续无上限扩张道路专用 DSL。

## 6. 核心术语

| 术语 | 规范定义 |
| --- | --- |
| Prompt | 一次或一类推理调用的外部上下文；可影响行为，但本身不构成可验证能力 |
| Experience | 从可观测轨迹和动作结果得到的事实记录，不包含隐藏真值 |
| Reflection | 对 Experience 的文本或结构化总结；默认仍属于 prompt 输入 |
| Candidate Skill | Agent 提出且通过 schema 校验，但尚未通过独立效果验证的可执行技能 |
| Verified Skill | 在隔离的未见环境中通过预注册门槛的技能版本 |
| Skill Set | 当前 Agent 可调用的版本化 Skill 集合及其验证证据 |
| Capability Score | 固定未见基准任务上，启用当前 Verified Skill Set 后得到的加权能力分数 |
| Environment Stage | 明确参数化的环境族，不是含义模糊的“难度数字” |
| Raw Percolation | 不施加可解性条件的拓扑样本，用于研究连通相变 |
| Conditioned Task | 不修改拓扑，仅在同一连通分量内放置任务对象的可解 Agent 任务 |

## 7. 当前基线边界

现有系统中的 `AgentMemory + reflection + prompt` 是经验驱动的提示适应；它不是 Verified Skill Set。现有 `RoadLearningModule` 是人工定义的、基于历史收益的估计模块，可作为 Handcrafted Skill 基线，但不得改称“Agent 自主发现的 Skill”。

任何研究报告必须保留上述边界，直到 M4 验收通过。
