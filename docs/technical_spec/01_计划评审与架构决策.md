# 7.14 计划评审与架构决策

版本：1.0  
状态：已评审  
评审对象：`docs/7.14计划.md`

## 1. 评审结论

结论：**符合导师建议，可以进入工程规格化阶段。**

计划不是简单扩大现有 demo，而是把研究对象提升为“可控环境分布如何驱动 Skill Set 的形成、验证、积累和迁移”。这与导师提出的大地图、分形布朗曲面、Hurst 指数、渗流模型以及 Agent/环境耦合图一致。

在工程实施前必须接受本文件中的九项修正。修正不会改变研究方向，只用于消除混淆变量、实现风险和不成立的因果结论。

## 3. 必须修正的设计点

### 3.1 H 不能同时含糊地代表所有空间结构

拓扑开放/障碍分布和通行成本分布必须是不同图层。两者可以共享 H 以做耦合实验，也必须支持独立随机场以做消融。

规范参数至少包括：

- `topology.model`
- `topology.p_open`
- `topology.hurst`
- `terrain.hurst`
- `terrain.roughness_strength`
- `resources.distribution`
- `resources.hurst`

若实验只报告一个 H，必须明确该 H 作用于哪些图层。

### 3.2 p 的语义必须唯一

本项目统一定义 `p_open` 为目标可通行格子比例。不得在部分代码中把 p 当障碍概率、在另一部分当开放概率。

生成器必须同时输出：

- `target_p_open`
- `realized_p_open`
- `largest_component_fraction`
- 横向/纵向贯穿标记

### 3.3 相关渗流临界点必须实测

独立四邻域方格 site percolation 的临界值可作为参考，但 fBm 阈值化后格点具有空间相关性，临界点会随 H、尺寸、边界和生成算法变化。

因此系统必须估计有限尺寸贯穿曲线：

```text
P_span(p_open | H, size)
```

不得把某个理论常数硬编码成课程边界。

### 3.4 不得用人工保护路径研究渗流

现有随机地图会保护基地到矿点的路径，这适合机制测试，不适合估计连通临界行为。新生成器必须支持：

1. `raw`：不保证任务可解，用于地图统计；
2. `conditioned_same_component`：不修改拓扑，在同一连通分量内放置基地和矿点；
3. `legacy_carved`：仅用于与旧实验兼容，禁止用于渗流结论。

### 3.5 大地图的每步复杂度必须与全图面积解耦

当前环境在构造 `info` 时会反复扫描全图并计算最短路。新架构必须：

- reset 后缓存静态地图诊断；
- 每步只更新动态增量指标；
- partial observation 的观测成本只依赖局部视野；
- 全图渲染和全图诊断只在显式请求时运行。

### 3.6 Skill 必须是受限可执行对象

一段反思文本、自然语言规则或详细 prompt 不等于 Skill。Skill 必须具备：

- 稳定 ID 和版本；
- 适用条件；
- 受限 DSL 表示的执行过程；
- 预算和终止条件；
- 目标指标；
- 来源轨迹；
- 验证证据；
- 可禁用、重放、迁移和消融的接口。

Agent 可以提出 DSL 的组合，但不得生成任意可执行代码。

### 3.7 Skill 晋升权必须属于独立验证器

Proposer、Solver 和 Verifier 必须逻辑隔离：

- Proposer 只能创建 Candidate；
- Solver 只能尝试 Candidate；
- Verifier 使用未见 seed 做配对测试；
- Registry 只接受 Verifier 签发的验证报告；
- LLM 文本不得直接设置 `status=verified`。

### 3.8 Skill 数量不是能力本身

能力图至少同时报告：

1. `verified_skill_count`：离散阶梯；
2. `capability_score`：固定未见任务族上的加权表现；
3. `skill_coverage_matrix`：技能对环境族的覆盖；
4. `false_trigger_rate`：负场景误触发；
5. `retention_score`：环境变化后的保持情况。

阶梯只能在新 Skill 通过验证后上升。多个 seed 取平均后曲线可能平滑，不应强行画成理想阶梯。

### 3.9 Chunk 不能使用 Python 内置 hash

Python 内置 `hash()` 不保证跨进程和跨版本稳定。区块种子必须通过稳定序列化和 BLAKE2b/SHA-256 派生。

独立对每个 chunk 做 FFT fBm 会产生边界断裂和统计不一致。Chunk 阶段必须使用全局坐标随机场或有可验证 halo 的算法，并检查：

- 相邻边界连续；
- 生成顺序无关；
- 重启后完全复现；
- 远离后重载仍保留动态改造。

## 4. 架构决策记录

### ADR-001：先有限大地图，后 chunk 开放世界

决定：M1 使用 64、128、256 等有限网格完成 H、p 和 size 校准；M7 才实现 chunk。

理由：如果有限地图上无法证明参数真的控制目标统计量，chunk 只会放大不可解释性和调试成本。

### ADR-002：静态生成层、动态改造层、对象层分离

决定：世界状态至少区分：

- 拓扑层：可通行/障碍；
- 连续地形层：`roughness in [0,1]`；
- 对象层：基地、矿石；
- 动态改造层：道路、挖通、资源耗尽；
- Agent 记忆层：仅保存已观测投影。

理由：当前单一 `Tile` 同时承担地形、对象和改造语义，无法自然表达连续成本和持久变更。

兼容策略：M1/M2 保留 `grid` 作为交互投影，同时增加 roughness sidecar；旧地图仍按离散成本运行。

### ADR-003：地图生成结果必须携带 provenance 和 diagnostics

决定：新生成器返回 `MapBuildResult`，不得只返回 grid。

理由：没有生成参数、seed、算法版本和诊断值，就无法复现实验或判断参数是否生效。

### ADR-004：Skill 使用声明式 DSL

决定：Skill procedure 只能由白名单操作符、白名单特征和结构化控制流组成。

理由：这既允许 Agent 自主组合行为，又能保证执行、审计、复现和消融；自然语言描述只作说明，不参与执行。

### ADR-005：Prompt 固定并记录哈希

决定：每个实验组必须声明 `base_prompt_id`、`base_prompt_hash` 和是否包含详细策略教学。

理由：否则无法判断能力提升来自 Skill，还是来自实验中偷偷改变的 prompt。

### ADR-006：验证采用未见地图上的配对消融

决定：同一 candidate 在相同 map seed、agent seed 和预算下分别启用/禁用，比较成对差值。

理由：地图方差大，非配对比较会浪费样本并产生虚假提升。

### ADR-007：能力分数来自固定基准，不来自训练窗口

决定：课程升级使用冻结的 gate benchmark；最终能力使用独立 test benchmark。

理由：训练 reward 同时受环境难度影响，不能作为跨阶段能力纵轴。

### ADR-008：自适应课程不得读取最终测试集

决定：课程控制器只能读取训练和 gate 分区，最终 test 仅在预定检查点运行。

理由：防止环境生成器对测试集过拟合。

### ADR-009：旧 road-learning 是 Handcrafted Skill 基线

决定：保留 `RoadLearningModule`、受控走廊和真实 LLM 结果，但在新实验矩阵中标记为 `handcrafted_skill`。

理由：它证明了经验归因可以影响后续修路，但其特征、阈值和研究对象由研究者预先规定，不能支持“自主发现 Skill”的强结论。

## 5. 目标系统边界

```text
Map Config + Seed
        |
        v
Fractal/Percolation Generator ---> Static Diagnostics
        |
        v
Layered Mutable Environment ---> Observation Projection ---> Agent Memory
        ^                                                   |
        |                                                   v
Environment Mutations <--- Skill Runtime <--- Skill Selector/Policy
                                                |
Trajectory + Outcome ----------------------------+
        |
        v
Skill Proposer -> Candidate Registry -> Independent Verifier
                                      |
                                      v
                              Verified Skill Registry
                                      |
                                      v
                         Frozen Capability Benchmark
```

环境层不依赖 LLM；Verifier 不信任 Proposer；Prompt 不直接执行动作；Skill runtime 不读取隐藏地图；最终能力评估不参与课程调参。

## 6. 现有代码迁移映射

目标架构是在当前代码上渐进扩展，不是推倒重写：

| 现有模块 | 保留价值 | 目标变化 |
| --- | --- | --- |
| `evogrid/envs/map_builder.py` | fixed/random/controlled-corridor 兼容入口 | 增加 generator registry 和 `build_map()`；旧函数保留适配 |
| `evogrid/envs/map_state.py` | 可变 grid、道路和挖掘记录 | 增加 roughness sidecar、map_id 和静态诊断缓存 |
| `evogrid/envs/evogrid_mine_env.py` | 动作语义、局部观测、统一 step | 分离 agent info/audit info，连续成本，移除每步全图诊断 |
| `evogrid/envs/road_credit.py` | 修路成本与后续复用归因 | 改为接收实际连续移动成本，保持记录结构可兼容 |
| `evogrid/agents/memory.py` | 只从观测和结果更新 | 增加 terrain band 和 Skill 使用结果，不引入隐藏真值 |
| `evogrid/agents/road_learning.py` | 已验证的经验估计机制 | 标记为 Handcrafted Skill 基线，不作为自主 Skill 实现 |
| `evogrid/agents/self_evolution_agent.py` | LLM、memory、reflection 和 fallback | 后续通过 Skill selector/runtime 组合，不直接承担 Registry/Verifier |
| `evogrid/llm/prompts.py` | 结构化动作 prompt | 固定模板版本和 hash；Skill proposal 使用独立 prompt 模块 |
| `scripts/run_generalization_eval.py` | 当前最完整的 train/test 评估流程 | 保留旧实验；通用 verifier、统计和 agent 类迁移到包内模块 |
| `tests/` | 旧行为回归保护 | 在不删除旧测试的基础上增加生成、泄漏、Skill 和统计测试 |

迁移期间禁止把现有 `RoadLearningModule` 重命名为 Self-proposed Skill 来绕过 M3/M4。它可以被包装成符合 Skill runtime 的人工 fixture，用于证明基础设施正确。

## 7. 可支持的研究结论层级

| 完成阶段 | 可以声称 | 不可以声称 |
| --- | --- | --- |
| M1 | H、p、size 可控地改变地图统计结构 | Agent 已自进化 |
| M2 | 连续地形和改造收益形成可审计因果链 | Skill 已自主形成 |
| M3 | Skill 可执行、持久化、组合和消融 | Skill 有效或可泛化 |
| M4 | 某 Candidate 在未见环境通过验证 | 整体 Skill Set 可持续增长 |
| M5 | 特定环境课程下 Skill Set 和能力发生变化 | 任意开放世界均适用 |
| M6 | 自适应课程维持了可测学习压力 | 存在普遍最优课程 |
| M7 | 技能可在可扩展持久世界中复用 | 理论意义上的无限世界已完全解决 |

## 8. 理论参考边界

实现采用以下理论作为生成和校准依据，但实验结论必须以本系统实测为准：

- Mandelbrot, B. B. & Van Ness, J. W. (1968), Fractional Brownian Motions, Fractional Noises and Applications, DOI: `10.1137/1010093`。
- Weinrib, A. & Halperin, B. I. (1983), Critical phenomena in systems with long-range-correlated quenched disorder, DOI: `10.1103/PhysRevB.27.413`。
- 独立方格 site percolation 的阈值只作为 sanity reference；相关场阈值和有限尺寸效应由 M1 校准实验决定。
