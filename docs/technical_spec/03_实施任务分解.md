# EvoGrid 智能体自进化平台：实施任务分解

版本：1.0  
状态：待实施  
规范来源：`02_可执行技术规格书.md`

## 1. 使用方式

本文件是底层模型的工作队列。每个工作包（WP）必须按顺序完成，并满足退出门槛后才能进入依赖它的工作包。

执行一个 WP 时必须：

1. 阅读该 WP 的输入文件和依赖；
2. 先确认工作区已有改动，不覆盖用户文件；
3. 只修改声明范围内的文件；
4. 为新增行为补测试；
5. 先运行定向测试，再运行完整测试；
6. 在输出中报告完成项、测试结果和未解决风险；
7. 不因“后面会用到”提前实现未授权里程碑。

基础验证命令：

```powershell
python -m unittest discover -s tests -p "test_*.py"
```

如果仓库后续引入其他测试框架，应更新本文件和 CI，不得只在本地口头约定。

## 2. 发布切分

| 发布 | 包含 | 研究产出 |
| --- | --- | --- |
| R1 地图统计版 | M0-M1 | 证明 p、H、size 可控地图结构 |
| R2 连续地形版 | M2 | 建立 roughness -> cost -> reward/road payoff 因果链 |
| R3 Skill 基础设施版 | M3-M4 | 区分 Prompt、Candidate 和 Verified Skill |
| R4 耦合实验版 | M5 | 生成 Agent/环境演化曲线和对照结果 |
| R5 自适应课程版 | M6 | 研究持续学习压力 |
| R6 开放世界版 | M7 | 持久、可扩展的 chunk 世界 |

R1 是下一阶段默认优先级。R1 未通过时不得开始 M3-M7。

## 4. M1：有限参数化地图

### WP-101：二维谱随机场

目标：实现 `spectral_fbm_v1`。

新增：

- `evogrid/envs/map_generation/spectral_field.py`
- `tests/test_spectral_field.py`

接口：

```python
def generate_rank_normalized_field(
    shape: tuple[int, int],
    hurst: float,
    seed: int,
) -> "np.ndarray":
    ...
```

实现要求：

- `float64` 计算，输出可转 `float32` 存储；
- 使用 `numpy.random.Generator`；
- 使用 `rfft2/irfft2`；
- DC 为 0；
- rank 归一化稳定；
- 不使用全局 `np.random.seed`；
- 不加入 SciPy 依赖。

测试：

- 形状和值域；
- 同 seed 一致；
- 不同 seed 不同；
- H 参数非法时报错；
- 非正方形地图；
- H=0.2/0.8 的经验空间相关统计方向正确；
- 无 NaN/Inf。

退出门槛：全部定向测试通过；生成算法和公式与规格一致。

### WP-102：拓扑与连通分量

目标：生成 iid/相关 site percolation 拓扑，不人工开路。

新增：

- `evogrid/envs/map_generation/connectivity.py`
- `tests/test_percolation_topology.py`

接口建议：

```python
def generate_open_mask(config, seed: int) -> np.ndarray:
    ...

def label_components(open_mask: np.ndarray) -> ComponentIndex:
    ...
```

要求：

- 四邻域；

- 支持 256x256；
- 不递归遍历可能导致 Python recursion overflow。

测试：

- `p_open=1` 全开放；
- 小型手工 mask 的组件数量和贯穿判断；
- correlated 实现的 realized p 精确到一个格子；
- raw 模式不会改变 mask；
- 相同 seed 一致。

- WP-708 小地图 Skill 到 chunk 世界迁移实验。

M7 必须额外测试：

- 从北到南和从南到北生成得到同一世界；
- 相邻 chunk 边界值连续；
- 重启后同坐标完全一致；
- 修路、挖掘、资源耗尽在卸载后仍保留；
- 未生成 chunk 不占用与世界面积线性增长的内存；
- Agent 仍只看到局部信息。

## 11. 禁止合并的任务

以下工作不得在同一个未经中间验收的大改动中完成：

- 新地图生成 + Skill runtime；
- 连续 reward + 自适应课程；
- Prompt 重写 + Skill 对照实验；
- Chunk 世界 + fBm 算法替换；
- Verifier gate 修改 + 正式实验复跑；
- 训练/verify/test seed 分区修改 + 结果汇总。

原因是这些组合会让回归来源和因果解释不可辨认。
