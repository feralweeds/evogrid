# EvoGrid 智能体自进化平台：可执行技术规格书

版本：1.0  
状态：生产实现基线  
依赖：`00_文档索引与执行规则.md`、`01_计划评审与架构决策.md`

## 1. 目标与范围

本规格定义从当前 EvoGrid-Mine 演进到“参数化环境 + 可验证 Skill Set”平台所需的接口和行为。

本期架构必须支持：

1. 由 `size`、`p_open`、Hurst 指数和资源分布控制的有限二维地图；
2. 拓扑层、连续通行成本层、对象层和动态改造层的分离；
3. 局部观测下无隐藏真值泄露的 Agent 交互；
4. 可持久化、可消融、可审计的 Candidate/Verified Skill；
5. Prompt-only、Handcrafted Skill、Self-proposed Skill 的严格对照；
6. 固定环境族、离散课程和后续自适应课程；
7. 生成、训练、验证、测试过程的完全可复现。

本期不要求：

- 立即实现无限 chunk 世界；
- 允许 LLM 生成任意代码；
- 立即证明 Skill Set 会无限增长；
- 用一个标量完全概括多维环境复杂度；
- 删除或替换现有 road-learning 实验。

## 3. 地图生成配置契约

### 3.1 YAML 结构

首个生产配置必须采用以下结构；字段名是规范接口：

```yaml
env:
  map_mode: fractal_percolation
  grid_size: [128, 128]
  max_steps: 3000
  world:
    schema_version: 1
    generator_version: spectral_fbm_v1
    world_seed: 0

    topology:
      model: correlated_site        # iid_site | correlated_site
      p_open: 0.65                  # 目标可通行比例
      hurst: 0.50                   # correlated_site 时必填，0 < H < 1
      solvability_mode: conditioned_same_component
      task_component: largest       # largest | sampled
      min_task_component_fraction: 0.20
      boundary_mode: finite_4_neighbor

    terrain:
      model: fractional_gaussian_surface
      hurst: 0.70
      base_move_cost: 0.01
      roughness_strength: 0.04
      cost_exponent: 1.0
      road_move_cost: 0.00
      observation_bins: [0.25, 0.50, 0.75]

    resources:
      distribution: clustered      # uniform | clustered
      count: 4
      hurst: 0.70                   # clustered 时必填
      renewable: true
      min_base_distance: 24
      min_pair_distance: 8

    placement:
      base_margin: 4
      max_attempts: 100

    diagnostics:
      enabled: true
      estimate_hurst: true
      articulation_points: true

  observation:
    mode: partial_obs
    local_view_radius: 4
    expose_continuous_roughness: false

  shaping:
    allow_dig: true
    allow_build_road: true
    reset_after_dropoff: false

  reward_mode: continuous_terrain
  rewards:
    dropoff: 10.0
    dig: -0.2
    build_road: -0.1
    mine: -0.01
    dropoff_action: -0.01
    noop: -0.01
    invalid_action: -0.05
```

### 3.2 配置校验

加载时必须执行以下校验，并以 `ValueError` 给出字段路径：

- `grid_size` 两维均大于等于 8；
- `0 < p_open <= 1`；
- 所有 H 满足 `0 < H < 1`；
- `base_move_cost >= 0`；
- `roughness_strength >= 0`；
- `cost_exponent > 0`；
- `observation_bins` 严格递增且位于 `(0,1)`；
- `resources.count >= 1`；
- 距离和 margin 非负；
- `raw`、`conditioned_same_component`、`legacy_carved` 以外的 solvability mode 必须拒绝；
- `legacy_carved` 必须在 manifest 中标记 `valid_for_percolation_analysis=false`。

不得静默截断、交换或修正非法参数。

### 3.3 Seed 语义

`env.reset(seed=x)` 中的 `x` 是本次地图的权威 `world_seed`。仅当 reset seed 为 `None` 时使用配置中的 `world_seed`。

不同随机子系统必须使用独立稳定子流：

```text
derive_seed(world_seed, "topology")
derive_seed(world_seed, "terrain")
derive_seed(world_seed, "resources")
derive_seed(world_seed, "placement")
derive_seed(world_seed, "agent")
```

`derive_seed` 必须：

1. 将输入编码为 UTF-8 的规范 JSON；
2. 使用 `hashlib.blake2b(..., digest_size=8)`；
3. 按无符号 64 位整数解析；
4. 不使用 Python 内置 `hash()`；
5. 有固定测试向量，保证重启和调用顺序不改变结果。

## 4. MapBuildResult 数据契约

新生成器必须返回 `MapBuildResult`。建议定义如下：

```python
@dataclass
class MapBuildResult:
    schema_version: int
    map_id: str
    grid: list[list[int]]
    roughness: list[list[float]]
    base_pos: tuple[int, int]
    ore_positions: set[tuple[int, int]]
    diagnostics: dict
    provenance: dict
```

约束：

- `grid` 是当前环境兼容用的交互投影；
- `roughness` 与 grid 同尺寸，值位于 `[0,1]`；
- 障碍格也保留潜在 roughness，DIG 后暴露该成本；
- `map_id` 由规范化参数、seed 和 generator version 的内容哈希产生；
- `diagnostics` 不得默认暴露给 partial-observation Agent；
- `provenance` 至少包含完整生成配置、world seed、子流 seed、生成器版本和代码版本（若可取得）；
- 数组写入 JSON 时使用普通 list；内部计算可以使用 NumPy 数组。

兼容入口：

```python
def build_map(config: dict, seed: int | None = None) -> MapBuildResult:
    ...

def build_fixed_map(config: dict | None = None, seed: int | None = None):
    result = build_map(config or {}, seed)
    return result.grid, result.base_pos, result.ore_positions
```

旧测试继续调用 `build_fixed_map`；新环境集成逐步改用 `build_map`。

## 5. 分形随机场生成算法

### 5.1 `spectral_fbm_v1`

有限地图参考实现必须只依赖 NumPy，并采用可复现谱滤波：

1. 用独立 RNG 生成二维标准高斯白噪声 `w`；
2. 计算 `rfft2(w)`；
3. 构造频率半径 `q = sqrt(kx^2 + ky^2)`；
4. 目标二维 fBm-like 表面的功率谱满足 `S(q) proportional q^-(2H+2)`；
5. 因此对白噪声频谱乘振幅滤波器 `A(q)=q^-(H+1)`；
6. 将 DC 分量设为 0；
7. `irfft2` 得到实数场；
8. 按稳定 rank 映射到 `[0,1]`，保留空间次序并固定边际分布。

若实现使用 `q2 = kx^2 + ky^2`，等价滤波器为：

```text
A(q2) = q2^(-(H+1)/2)
```

实现不得把多 octave Perlin noise 直接标记为严格 fBm。若后续替换算法，必须更改 `generator_version` 并重新校准。

### 5.2 统计校准

每个生成版本必须验证：

- 同 seed、同参数逐元素一致；
- 不同 seed 的场不完全相同；
- rank 映射后均值接近 0.5；
- 经验半变异函数在预注册 lag 区间内的 log-log 斜率随 H 单调增加；
- H=0.2、0.5、0.8 的估计分布具有可分性；
- 旋转或轴方向不出现明显系统偏差。

经验 H 只用于诊断，不反向修改生成地图。

## 6. 拓扑生成与可解性

### 6.1 独立渗流

`topology.model=iid_site` 时，每格通过独立均匀随机数与 `p_open` 比较决定开放状态。实际开放率自然波动，必须记录 `realized_p_open`。

### 6.2 相关渗流

`topology.model=correlated_site` 时：

1. 使用 topology 子流生成独立相关场；
2. 按 rank 选择恰好 `round(p_open * cell_count)` 个开放格；
3. rank 相同的极小概率情况用行优先索引稳定打破；
4. 不得在阈值化后人工开路。

H 控制空间相关，`p_open` 控制边际开放比例。二者必须可以独立改变。

### 6.3 三种可解性模式

`raw`：

- 不修改拓扑；
- 基地和矿石可以不连通；
- 用于贯穿概率和连通簇统计；
- Agent 任务默认不得使用，除非研究不可解检测。

`conditioned_same_component`：

- 不修改拓扑；
- 在指定连通分量内放置基地和任务矿石；
- 若最大分量小于配置门槛，可以生成但标记 `task_placement_failed`，也可以由调用方跳过；
- 跳过和重采样次数必须记录，不得伪装成原始分布。

`legacy_carved`：

- 允许旧算法保护或开出路径；
- 仅用于兼容旧 road-learning；
- 输出必须标记不能用于渗流临界结论。

## 7. 资源生成

`uniform`：在允许的任务连通分量内均匀选择，满足最小距离。

`clustered`：

1. 使用 resources 子流生成独立相关场；
2. 只在允许的开放候选格上排序；
3. 按高值优先并满足 `min_pair_distance` 选择矿点；
4. 不得使用 terrain/topology 的同一随机场，除非实验配置显式声明 `couple_fields=true`；
5. `couple_fields=true` 必须作为实验因素记录。

资源可再生性由 `renewable` 控制。为保持当前重复运输实验，默认 `true`。有限资源模式必须记录 depleted 状态，且不得改变旧配置行为。

## 8. 地图诊断契约

生成后一次性计算并缓存静态诊断。至少输出：

```json
{
  "schema_version": 1,
  "map_id": "...",
  "target_p_open": 0.65,
  "realized_p_open": 0.6500,
  "component_count": 0,
  "largest_component_size": 0,
  "largest_component_fraction": 0.0,
  "spans_horizontal": false,
  "spans_vertical": false,
  "base_ore_reachable_fraction": 0.0,
  "shortest_path_length": null,
  "minimum_cost_path_cost": null,
  "path_tortuosity": null,
  "articulation_point_count": 0,
  "roughness_mean": 0.0,
  "roughness_std": 0.0,
  "rough_patch_count": 0,
  "largest_rough_patch_fraction": 0.0,
  "estimated_terrain_hurst": null,
  "estimated_topology_hurst": null,
  "valid_for_percolation_analysis": true,
  "placement_status": "ok"
}
```

定义：

- 连通使用四邻域；
- 横向贯穿表示存在开放簇连接最左和最右边界；
- 纵向贯穿定义类似；
- `path_tortuosity = shortest_path_length / manhattan_distance`；
- 最低成本路径使用未改造初始 terrain cost；
- articulation point 只在任务连通分量上统计；
- rough patch 使用最高 roughness 档位的四邻域连通分量；
- null 与 0 必须区分：无路径为 null，不是长度 0。

性能约束：静态诊断只能在 reset、显式审计请求或 episode 结束时运行，禁止每个 step 全图重算。

## 9. 连续地形与环境集成

### 9.1 MapState

`MapState` 必须增加：

```python
roughness: list[list[float]] | None = None
map_id: str = ""
static_diagnostics: dict = field(default_factory=dict)
```

旧地图的 `roughness=None` 走 legacy discrete reward。新地图必须携带 roughness。

### 9.2 移动成本

`reward_mode=continuous_terrain` 时，未修路格子的移动成本为：

```text
C_move(r) = base_move_cost + roughness_strength * r^cost_exponent
reward_move = -C_move(r)
```

道路格子的移动成本为 `road_move_cost`。修路前必须记录该位置的原始成本，RoadCredit 使用实际节省：

```text
saved_per_use = original_move_cost - road_move_cost
road_net_payoff = usage_count * saved_per_use - build_cost
```

不得同时叠加旧 `move_ground/move_rough` 惩罚和连续成本。

`reward_mode=legacy_discrete` 时，行为完全保持现状。

### 9.3 DIG 语义

DIG 将动态拓扑从 blocked 改为 open，但不把 roughness 强制设为 0。被挖开的格子使用生成时已存在的潜在 roughness。

### 9.4 观测投影

默认不向 Agent 暴露连续浮点粗糙度，而按 `observation_bins` 投影为：

```text
[0, b0)      -> SMOOTH
[b0, b1)     -> NORMAL
[b1, b2)     -> ROUGH
[b2, 1]      -> VERY_ROUGH
```

观测中可以继续提供兼容 tile，但必须增加明确的 `terrain_band`。不得仅靠 Tile 枚举猜测连续成本。

只有 `expose_continuous_roughness=true` 的专门消融组可以看到精确值。

### 9.5 Agent info 与审计 info 分离

partial observation 下，返回给 `agent.act(obs, info)` 的 info 只能包含可由自身动作和历史观测得到的信息。

以下内容不得进入 Agent info：

- 全图连通分量；
- 隐藏矿石坐标或数量；
- 隐藏最短路；
- 全图 route rough count；
- 未来道路真实使用次数；
- 当前格是否位于隐藏最优路线。

环境必须提供单独的 evaluator-only 接口：

```python
def get_audit_snapshot(self) -> dict:
    ...
```

rollout/evaluation 可以在 episode 结束后读取，Agent 不得持有该返回值。

## 10. MapGenerator 接口

所有新生成器实现统一协议：

```python
class MapGenerator(Protocol):
    name: str
    version: str

    def build(self, config: MapGenerationConfig, seed: int) -> MapBuildResult:
        ...
```

`map_builder.py` 维护显式 registry。未知 `map_mode` 必须报错，不得回退到 fixed map。

生成器本身不得读取 Agent、Skill、LLM 或输出目录。

## 11. SkillSpec 数据契约

### 11.1 Skill 与文本说明分离

Skill 的 `description`、`rationale` 可由自然语言生成，但 runtime 只能读取结构化字段。建议 schema：

```json
{
  "schema_version": 1,
  "skill_id": "reusable_road_building",
  "version": "1.0.0",
  "status": "candidate",
  "name": "Reusable road building",
  "description": "Build only where repeated transport can repay construction.",
  "problem_addressed": "Repeated high-cost transport",
  "source": {
    "proposer": "llm",
    "source_episode_ids": ["run/episode/3"],
    "base_prompt_hash": "..."
  },
  "applicability": {
    "all": [
      {"feature": "current.terrain_band", "op": "in", "value": ["ROUGH", "VERY_ROUGH"]},
      {"feature": "route.is_known_transport_route", "op": "eq", "value": true}
    ]
  },
  "procedure": [
    {"op": "ESTIMATE", "estimator": "future_route_uses", "store_as": "n_use"},
    {"op": "ESTIMATE", "estimator": "road_break_even_uses", "store_as": "n_break_even"},
    {
      "op": "IF",
      "condition": {"left": {"var": "n_use"}, "op": "gte", "right": {"var": "n_break_even"}},
      "then": [{"op": "ACT", "action": "BUILD_ROAD"}],
      "else": [{"op": "RETURN", "result": "not_applicable"}]
    }
  ],
  "budget": {
    "max_runtime_steps": 4,
    "max_environment_actions": 1,
    "max_nested_skill_depth": 0
  },
  "objective": {
    "primary_metric": "road_net_payoff",
    "direction": "maximize",
    "negative_context_metric": "false_trigger_rate"
  },
  "dependencies": [],
  "spec_hash": "..."
}
```

### 11.2 状态机

唯一合法状态转换：

```text
proposed -> candidate
candidate -> verifying
verifying -> verified | revision_required | rejected
revision_required -> candidate
verified -> deprecated
deprecated -> candidate   # 必须生成新版本，不得原地复活
```

LLM/Proposer 只能输出 `proposed` 内容。Schema validator 创建 Candidate。Verifier 独占 `verifying` 后续状态写权限。

任何内容变化必须增加版本并产生新的 `spec_hash`。验证报告只对一个精确 hash 有效。

### 11.3 Predicate DSL

允许的逻辑节点：

- `all`
- `any`
- `not`
- 叶节点 `{feature, op, value}`

允许的比较操作：

- `eq`, `ne`
- `lt`, `lte`, `gt`, `gte`
- `in`, `not_in`

feature 必须来自 `SkillContext` 白名单，例如：

- `current.terrain_band`
- `current.tile_type`
- `cargo.has_ore`
- `route.exists`
- `route.is_known_transport_route`
- `route.remaining_length_bucket`
- `memory.visit_count_bucket`
- `memory.similar_outcome_count`
- `memory.similar_mean_payoff`
- `local.adjacent_obstacle_count`
- `local.frontier_count`

禁止特征：

- hidden shortest path；
- hidden ore position；
- future usage truth；
- map scenario label；
- test split label；
- evaluator-only diagnostics。

### 11.4 Procedure DSL

V1 白名单操作：

| op | 作用 | 约束 |
| --- | --- | --- |
| `ESTIMATE` | 调用注册的可观测估计器 | estimator 必须白名单注册 |
| `SELECT_TARGET` | 从记忆/局部观测选目标 | 不得访问隐藏 grid |
| `PLAN_ROUTE` | 基于 Agent memory 规划 | 必须标记 unknown-cell policy |
| `IF` | 结构化条件分支 | 嵌套深度默认不超过 4 |
| `ACT` | 发出一个 primitive action | 仍经过环境合法性过滤 |
| `FOLLOW_ROUTE` | 执行已生成路线的有限前缀 | 必须有 max_steps |
| `CALL_SKILL` | 调用依赖 Skill | 依赖图无环，深度受限 |
| `RETURN` | 返回结果 | 必须可序列化 |

V1 不允许循环。重复行为通过多次事件触发，而不是 DSL 中的无界循环。

### 11.5 SkillContext

Runtime 每次调用接收：

```python
@dataclass
class SkillContext:
    observation: dict
    observable_info: dict
    memory_summary: dict
    route_plan: dict | None
    episode_budget: dict
```

构造 SkillContext 的组件必须通过 hidden-truth leakage 测试。

## 12. Skill Runtime

Skill runtime 必须满足：

1. 输入 SkillSpec、SkillContext 和注册的 estimator/controller；
2. 执行前再次校验 spec hash 和状态；
3. Candidate 只有实验明确允许时可执行；正常策略只加载 Verified；
4. 每个操作符写 trace；
5. 超预算、未知操作符、未知 feature、非法动作时安全终止；
6. 不执行 description/rationale 中的文字；
7. 不调用 `eval`、`exec`、动态 import 或 shell；
8. nested Skill 调用必须无环；
9. 同输入、同 registry、同 seed 时结果可重放。

最少 trace：

```json
{
  "schema_version": 1,
  "run_id": "...",
  "episode_id": "...",
  "step": 42,
  "skill_id": "...",
  "skill_version": "1.0.0",
  "spec_hash": "...",
  "applicable": true,
  "operations": [],
  "chosen_action": "BUILD_ROAD",
  "termination": "completed",
  "observable_context_hash": "..."
}
```

## 13. Skill Proposer

Proposer 输入只能包含：

- 训练分区的可观测轨迹；
- 动作结果；
- 已有 Skill 的公开元数据；
- 允许的 DSL、feature 和 estimator 列表；
- 固定的 proposer prompt id/hash。

Proposer 输出必须先经过：

1. JSON 提取；
2. schema 校验；
3. 白名单校验；
4. 预算校验；
5. 依赖图校验；
6. 与现有 Skill 的结构去重；
7. 内容 hash 计算。

失败只产生 `proposal_rejected` 事件，不创建可执行 Skill。

不得在 proposer prompt 中直接要求“必须提出修路技能”。可以要求识别重复失败、效率瓶颈或可复用策略。

## 14. Skill Registry

Registry 必须区分物理存储：

```text
skills/
├── candidates/<skill_id>/<version>.json
├── verified/<skill_id>/<version>.json
├── deprecated/<skill_id>/<version>.json
├── reports/<skill_id>/<version>/<verification_id>.json
└── registry_events.jsonl
```

Registry API：

```python
register_candidate(spec) -> SkillRecord
begin_verification(skill_id, version) -> VerificationLease
apply_verification(report) -> SkillRecord
deprecate(skill_id, version, reason) -> SkillRecord
list_available(context, status="verified") -> list[SkillRecord]
```

`apply_verification` 必须检查：

- report 的 candidate hash 与当前 spec hash 一致；
- verify seeds 不与 source/train seeds 重叠；
- report schema 完整；
- 所有预注册 gate 已计算；
- report 的最终 decision 由确定性 gate 逻辑产生，而不是自由文本。

## 15. Prompt 与 Skill 隔离契约

每次 LLM 调用必须记录：

- `prompt_role`：action/planner/reflection/proposer；
- `prompt_template_id`；
- `prompt_template_version`；
- `prompt_hash`；
- 是否包含 detailed strategy instruction；
- 注入的 Skill ID 列表，但不在 Self-proposed+Verified 组注入详细执行教程。

实验组规则：

| 组 | Prompt | Skill runtime |
| --- | --- | --- |
| No Skill | 固定基础任务 prompt | 禁用 |
| Prompt-only | 固定详细策略 prompt | 禁用，跨回合不保存 Skill |
| Handcrafted Skill | 固定基础 prompt | 仅人工 Skill |
| Self-proposed Candidate | 固定基础 prompt | 仅允许受控尝试 Candidate |
| Self-proposed + Verified | 固定基础 prompt | 仅加载 Verified |
| Prompt + Skill | 详细策略 prompt | Verified，作为上限参考 |

更改 prompt version 视为新的实验条件，不得与旧条件直接合并。

## 17. Capability Score

Agent 能力不得由训练 reward 直接表示。平台必须保存固定任务族上的能力向量：

```text
capability_vector(S) = [score_task_1, ..., score_task_m]
```

标量仅用于画总览图：

```text
capability_score(S) = sum_j weight_j * normalized_score_j(S)
```

约束：

- 权重和归一化范围在实验前固定；
- 同一论文图中不得中途改变任务族或权重；
- 同时报告 `verified_skill_count`；
- 新 Skill 通过验证但导致 set-level capability 下降时，阶梯数量可以上升，但能力曲线必须如实下降；
- 必须保存完整 skill-by-task coverage matrix，不能只保留总分。

## 18. Curriculum 契约

课程阶段必须显式表示参数，而不是只有 difficulty：

```yaml
stages:
  - stage_id: open_small
    env_family:
      sizes: [[64, 64]]
      p_open: [0.75]
      topology_hurst: [0.5]
      terrain_hurst: [0.2, 0.5, 0.8]
    train_budget_episodes: 100
    gate_benchmark_id: open_small_gate_v1
    promotion_rule:
      capability_score_gte: 0.70
      consecutive_windows: 2
```

固定课程阶段只能读取 train/gate 分区。进入下一阶段必须基于冻结窗口，不得根据单回合 reward。

自适应控制器只能在 M5 完成后启用，且每次参数变化写 `curriculum_events.jsonl`：

```json
{
  "from_stage": "...",
  "to_parameters": {},
  "observable_evidence": {},
  "controller_version": "...",
  "reason_code": "too_easy|too_hard|learning_frontier"
}
```

## 20. 性能与复杂度要求

1. 有限谱生成应为 `O(HW log(HW))`；
2. reset 后的普通 step 不得扫描完整地图；
3. partial observation 构造应为 `O(radius^2)`；
4. 静态 diagnostics 在 reset 缓存；
5. 动态全图连通性只在显式 checkpoint 计算；
6. 256x256 地图必须可在普通 CPU 环境运行，不依赖 GPU；
7. full_obs 在大地图中只用于诊断，默认实验必须 partial_obs；
8. Skill runtime 单次执行受预算限制，不得有无界循环；
9. LLM 调用保持事件触发，不按每一步强制调用。

## 21. 错误处理

| 错误 | 必须行为 |
| --- | --- |
| 非法地图参数 | 立即报错，指出字段路径 |
| 地图放置失败 | 返回可诊断状态或受控异常，记录 seed，不静默换 seed |
| 未知 map mode | 报错，不回退 fixed |
| Skill schema 非法 | proposal rejected，不进入 registry |
| Skill 访问未知 feature | runtime 安全终止并记录 |
| Skill 超预算 | runtime 终止，回退基础策略 |
| LLM 输出非法 | 记录并 fallback，不改变 Skill 状态 |
| verify seed 泄漏 | verification invalid，禁止晋升 |
| manifest 写入失败 | run 标记 failed，不宣称实验完成 |
