# 下一阶段实验路线：从机制验证到可证伪的自适应学习实验

本文档记录 EvoGrid-Mine 当前实验状态、研究边界，以及下一阶段应该优先实现的实验路线。

核心判断：

- 当前项目不是无意义的手工演示。它已经有可变环境、局部观测、跨 episode 记忆、road payoff 反馈和 LLM 决策接口。
- 但它还不能强声称已经完成“完整 agent 自进化”或“在改变环境中学习”。
- 下一阶段的目标不是继续堆更复杂的 prompt，而是把实验改造成可泛化、可复现、可反证的研究设置。

## 1. 当前状态

### 已经成立的部分

项目目前已经具备几个重要组件：

1. 环境可被 agent 改造

   `DIG` 和 `BUILD_ROAD` 会真实修改 grid。环境会记录 `built_roads`、`dug_cells`、`changed_cells` 和道路收益。

2. partial observation 是真实约束

   `partial_obs` 只暴露 agent 周围局部窗口、当前位置、基地位置、携矿状态和已交付数量，不直接提供全图或隐藏矿点。

3. agent 有跨 episode 记忆

   `AgentMemory` 会保存见过的 tile、矿点、障碍、粗糙地形、失败动作、road credit records 和 reflections。

4. road learning 有反馈闭环

   agent 建路之后，环境统计道路使用次数、节省移动成本、建造成本和净收益。后续 `RoadLearningModule` 会用这些历史记录估计同类地形建路价值。

5. LLM road-learning 接口已经成形

   `LLMRoadLearningAgent` 会把 `ShapingOpportunity`、`learned_estimate`、route context 和 exploration budget 放进 prompt，让 LLM 决定是否执行 `BUILD_ROAD`。

### 还不能成立的强主张

当前还不应该声称：

1. agent 已经完成完整自进化。
2. agent 已经在开放变化环境中学会环境塑造。
3. LLM 自主发现了稳定可泛化的建路策略。
4. 当前 road-learning 结果可以直接代表真实复杂地图中的学习能力。

原因：

- 关键 road-learning 环境仍然是小型手工走廊。
- 地图生成基本是 fixed curriculum。
- agent 没有参数级学习，主要依赖记忆、阈值估计、反思文本和 prompt 条件变化。
- 大图 self-evolution 可以探索和找矿，但目前没有稳定学会建路。
- 还缺少 train/test 地图分离、随机地图、多 seed 统计和非平稳扰动。

## 2. 下一阶段总目标

把项目从：

```text
固定地图上的机制验证
```

推进到：

```text
随机地图 + 泛化评估 + 非平稳扰动 + 严格 baseline 的自适应学习实验
```

下一阶段的核心研究问题：

> 在只拥有局部观测和自身历史经验的条件下，agent 是否能通过探索和反馈，在未见过的变化地图中学会什么时候改造环境是有收益的？

## 3. 优先级最高的任务

### 任务 A：随机地图生成器

这是下一阶段最重要的基础设施。

当前 `configs/env_road_learning.yaml` 是一个 5 x 16 粗糙走廊。它适合验证 road payoff 机制，但太容易被解释为人为规划。

下一步应支持随机地图配置：

```yaml
env:
  map_mode: random_curriculum
  grid_size: [16, 16]
  max_steps: 400
  base_pos: [2, 2]
  random_map:
    ore_count: 1
    min_base_ore_distance: 10
    obstacle_density: 0.12
    rough_density: 0.25
    rough_corridor_bias: 0.4
    ensure_reachable: true
    max_generation_attempts: 100
  observation:
    mode: partial_obs
    local_view_radius: 4
  shaping:
    allow_dig: true
    allow_build_road: true
```

生成器必须满足：

- 可通过 seed 复现。
- 基地和矿点之间至少存在一条可达路径。
- rough terrain 不应只出现在固定直线走廊。
- obstacle 不能把地图切成不可解。
- 支持多个矿点，但第一阶段可以先做单矿点。
- full_obs 只能用于 oracle baseline 和地图验证，不能进入正式 agent prompt。

建议文件：

- 修改 `evogrid/envs/map_builder.py`
- 新增 `configs/env_random_curriculum.yaml`
- 新增 `tests/test_random_map_builder.py`

验收标准：

- 对 seeds `0..49` 生成地图全部可达。
- 同一 seed 生成完全一致地图。
- 不同 seed 地图有明显差异。
- partial_obs 中不泄露隐藏矿点全局坐标。

### 任务 B：泛化评估脚本

仅在训练地图上变好不够。必须做 train/test split。

建议新增脚本：

```text
scripts/run_generalization_eval.py
```

推荐实验设置：

```text
train seeds: 0..49
test seeds: 1000..1049
episodes per seed: 3
max steps: 400
```

第一版可以不做 PPO 或参数训练，而是让 memory/road learning 在 train seeds 上积累经验，再冻结或复制策略状态到 test seeds 上评估。

对比组：

1. `route_only`
2. `random_road`
3. `rough_rule_road`
4. `exploration_road`
5. `llm_no_road_learning`
6. `llm_with_road_learning`

核心指标：

- `ore_delivered`
- `episode_reward`
- `transport_steps_per_ore`
- `num_build_road`
- `road_net_payoff`
- `positive_road_ratio`
- `avg_payoff_per_road`
- `p_build_given_learned_positive`
- `p_build_given_learned_nonpositive`
- `test_minus_train_reward_gap`
- `test_minus_train_road_payoff_gap`

验收标准：

- 输出 `metrics.csv`、`summary.json`、`group_comparison.csv`。
- summary 中明确区分 train/test。
- 能回答：agent 是否只记住训练地图，还是学到了可迁移的建路判断？

### 任务 C：非平稳环境扰动

“在改变环境中学习”需要环境真的改变，而不是只换 seed。

建议先实现 episode-level perturbation：

```yaml
env:
  nonstationary:
    enabled: true
    schedule:
      - episode: 10
        type: road_cost_increase
        move_road: -0.02
      - episode: 20
        type: rough_density_shift
        rough_density_delta: 0.15
      - episode: 30
        type: ore_relocation
```

第一阶段不需要很复杂。最关键的是让 agent 面对过去经验失效的情况。

可选扰动：

- 道路移动收益下降。
- 建路成本上升。
- rough terrain 分布改变。
- 某些已建道路失效。
- 新矿点出现。
- 旧矿点枯竭。
- obstacle 局部变化。

新增指标：

- `adaptation_episode`
- `pre_shift_reward_mean`
- `post_shift_reward_mean`
- `recovery_steps`
- `stale_memory_error_count`
- `negative_transfer_count`

验收标准：

- shift 后 agent 的旧策略应该先受损。
- 若学习机制有效，若干 episode 后指标应恢复。
- road-learning agent 应减少继续执行已经变差的建路行为。

### 任务 D：合并 self-evolution 和 road-learning 主线

当前有两条线：

- 大图 partial_obs self-evolution：会探索、记忆和找矿，但不稳定建路。
- 小图 road-learning：会根据 road payoff 学建路，但环境过于手工。

下一阶段应该合并为：

```text
partial_obs
+ memory
+ route planner over observed map
+ exploration-based road sampling
+ RoadLearningModule
+ LLM-mediated shaping decision
+ randomized/nonstationary maps
```

目标不是让 LLM 从零承担所有低层动作，而是让 LLM 在有结构化证据时决定是否改造环境。

## 4. Baseline 设计

为了让实验可反证，必须加入反例 baseline。

### `route_only`

只寻路、采矿、返回，不建路。

作用：证明建路机制是否真的带来额外收益。

### `random_road`

在可建路位置以固定概率建路。

作用：证明 agent 不是因为“随便多建路”而变好。

### `rough_rule_road`

遇到 rough tile 就建路。

作用：对比简单规则是否已经足够。如果 LLM/learning agent 赢不了这个 baseline，自进化主张不成立。

### `exploration_road`

非 LLM 的探索阈值 agent。

作用：区分“学习机制有效”和“LLM 真带来额外价值”。

### `llm_no_road_learning`

LLM 可以看当前 opportunity，但看不到历史 road payoff。

作用：测试历史经验是否真的影响 LLM 决策。

### `llm_with_road_learning`

LLM 同时看到当前 opportunity 和历史 learned estimate。

作用：主实验组。

## 5. 反作弊约束

正式自适应实验中必须遵守：

1. partial_obs agent 不得读取完整 grid。
2. partial_obs agent 不得读取隐藏矿点坐标。
3. road-learning estimate 只能来自 agent 已经执行过或观察到的 road credit records。
4. `global_prior` 不能直接触发建路，只能作为背景统计。
5. oracle 只能作为 upper-bound sanity baseline，不得混入主实验策略。
6. warm-start 数据必须单独标记，不能和 cold-start 自学习结果混淆。
7. test seeds 不得用于调参。
8. LLM prompt 中必须明确标记 learned estimate 的证据数量和来源。

## 6. 推荐实现顺序

### 第 1 步：随机地图生成与测试

交付物：

- `configs/env_random_curriculum.yaml`
- `tests/test_random_map_builder.py`
- `map_builder.py` 支持 `map_mode: random_curriculum`

完成后先不接 LLM，只验证地图质量。

### 第 2 步：随机地图上的非 LLM baseline

交付物：

- `random_road` baseline
- `rough_rule_road` baseline
- `run_generalization_eval.py` 第一版

完成后先比较：

```text
route_only
random_road
rough_rule_road
exploration_road
```

### 第 3 步：接入 LLM road-learning

交付物：

- `llm_no_road_learning`
- `llm_with_road_learning`
- LLM trace 中记录 learned estimate、build source、rationale

目标是证明：

```text
p_build_given_learned_positive
>
p_build_given_learned_nonpositive
```

并且 road payoff 仍为正。

### 第 4 步：加入非平稳扰动

交付物：

- `configs/env_nonstationary_random.yaml`
- perturbation scheduler
- adaptation metrics

目标是观察旧经验失效后，agent 是否修正建路行为。

### 第 5 步：整理论文式实验结果

交付物：

- `outputs/runs/generalization_eval_*/summary.json`
- `outputs/runs/nonstationary_eval_*/summary.json`
- 曲线图
- 表格
- qualitative traces

## 7. 最小可接受实验矩阵

第一版不需要太大，避免成本失控。

```text
Map type:
  random_curriculum

Train:
  seeds 0..19
  episodes per seed 2

Test:
  seeds 1000..1019
  episodes per seed 2

Groups:
  route_only
  rough_rule_road
  exploration_road
  llm_no_road_learning
  llm_with_road_learning

Metrics:
  episode_reward
  ore_delivered
  road_net_payoff
  positive_road_ratio
  p_build_given_learned_positive
  p_build_given_learned_nonpositive
```

如果这组跑通，再扩大 seed 数。

## 8. 成功判据

下一阶段可以声称“更接近在改变环境中学习”，至少需要满足：

1. 在 test seeds 上，`llm_with_road_learning` 或 `exploration_road` 的 road payoff 为正。
2. 建路不是随机发生，而是更多发生在 learned estimate 为正的位置。
3. `llm_with_road_learning` 相比 `llm_no_road_learning` 有更高的 learned-positive 建路概率，且不会显著增加负收益道路。
4. 相比 `route_only`，主实验组在 reward 或 transport efficiency 上有稳定收益，或者至少 road_net_payoff 明确为正。
5. 在非平稳扰动后，agent 能减少旧策略导致的负迁移。

如果这些条件不满足，应诚实结论为：

```text
当前机制可以记录和利用环境改造反馈，但尚未形成可泛化的自适应环境塑造能力。
```

## 9. 失败时优先排查

如果随机地图上效果不好，优先检查：

1. 地图是否过难，partial_obs agent 找不到矿。
2. route planner 是否过弱，导致 road opportunity 很少出现在有效路径上。
3. road cost 是否设置不合理，使建路几乎永远不可能正收益。
4. exploration budget 是否过低，导致没有足够 road credit samples。
5. learned estimate 是否过窄，只按 tile type 聚合不够，需要加入 route context 特征。
6. LLM prompt 是否没有清楚表达历史收益证据。
7. baseline 是否已经很强，导致道路收益被步数成本抵消。

## 10. 研究表述边界

在下一阶段完成前，推荐对外表述为：

> EvoGrid-Mine 当前实现了一个部分可观测、可被 agent 改造的网格环境，并提供了基于记忆和 road payoff 的环境塑造学习机制。已有结果说明该机制能在受控场景中影响建路决策，但仍需要随机地图、泛化评估和非平稳扰动来支持更强的自适应学习主张。

完成随机化、泛化和非平稳实验后，如果结果满足成功判据，可以升级表述为：

> 在部分可观测、随机化且发生环境变化的地图中，agent 能基于自身历史环境改造反馈调整后续建路行为，并在未见地图上保持正向 road payoff 或任务效率收益。
